audit-log-service API 문서

audit-log-service는 append-only 감사 로그 저장과 관리자 조회를 담당합니다. 현재 NestJS 실행 구조, PostgreSQL 저장소, 이벤트 기반 저장 consumer, internal auth 기반 내부 조회 API가 구현되어 있습니다.

서비스 상태

상태이벤트 저장 구현 NestJS bootstrap, config, health/readiness, request context, access log, security headers, internal auth guard, append-only repository/service, 이벤트 consumer, 조회 API가 있습니다.
소유 범위감사 로그 저장, 조회, 보존 정책, evidence 추적
주요 연동저장은 auth-iam-service, wms-service, tenant-service의 outbox 이벤트를 소비합니다. 조회는 admin-bff-service가 내부 API로 호출합니다.
배포 상태CDK 스택에 ECR, ECS Fargate service, CloudWatch log group, 내부 SG, DB secret, audit internal auth secret 구성이 포함되어 있습니다.

Audit 저장 이벤트

업무 행위 완료 후 각 서비스는 같은 DB transaction 안에서 outbox event를 저장하고, relay가 EventBridge/SQS로 발행한 이벤트를 audit-log-service가 소비해 append-only 감사 로그로 기록합니다. AWS dev/staging/prod 기본 경로는 CDK 관리형 audit event SQS queue입니다.

ConsumerAUDIT_EVENT_CONSUMER_ENABLED=true이면 SqsAuditEventWorker가 AUDIT_EVENT_QUEUE_URL을 polling합니다.
성공 처리감사 로그 저장 성공 후에만 SQS message를 삭제합니다.
실패 처리저장 실패 시 message를 삭제하지 않아 SQS retry와 DLQ redrive 정책으로 처리합니다.

필수eventId, eventType, schemaVersion, tenantId, requestId, occurredAt, source, aggregateType, aggregateId, actor.type
중복 처리eventId는 audit_logs.event_id unique key로 저장합니다. 같은 이벤트가 재전달되면 기존 auditId를 반환하고 중복 row를 만들지 않습니다.
저장 매핑eventType은 audit action, aggregateType/aggregateId는 audit resource, data는 audit details로 저장합니다.
actor.typeuser, system, service
저장 금지data에는 password, token, Authorization, cookie, secret, API key, private key, SQL, stack trace를 포함할 수 없습니다.

응답과 오류 기준

Envelope모든 응답은 success, requestId, timestamp를 포함합니다.
List목록 응답은 data.items, data.page, data.size, data.total만 사용합니다.
검증 오류요청값 검증 실패는 공통 코드 VALIDATION_FAILED를 사용하고, error.details.fields는 필드명-메시지 객체로 내려줍니다.
Audit 전용 오류AUDIT_INTERNAL_AUTH_NOT_READY, AUDIT_INTERNAL_AUTH_REQUIRED, AUDIT_INTERNAL_SERVICE_FORBIDDEN, AUDIT_INTERNAL_SIGNATURE_INVALID, AUDIT_INTERNAL_TIMESTAMP_INVALID, AUDIT_INTERNAL_TIMESTAMP_EXPIRED, AUDIT_LOG_WRITE_FAILED, AUDIT_LOG_QUERY_FAILED
장애 정책저장 실패는 이벤트 consumer/relay의 retry와 DLQ 대상으로 처리합니다. 원 업무 API는 audit-log-service HTTP 응답에 직접 결합하지 않습니다. SQS consumer는 성공 메시지만 삭제합니다.

오류 코드 매핑

HTTPCode상황
400VALIDATION_FAILED필수 필드 누락, UUID/date/page/result enum 오류, details 민감 키 포함, from이 to보다 늦은 경우
