gateway-service API

Proxy Routes

인증된 /api/** 요청에 request context를 붙이고 내부 서비스로 전달합니다.

ANY /api/auth/** /api/admin/** /api/app/** ACTIVE

기본 정보

인증필수. Authorization: Bearer {token}이 필요합니다. 단, 로그인과 토큰 갱신 endpoint는 JWT 없이 auth-iam-service로 전달합니다.
Public 예외/health, /ready, POST /api/auth/login, POST /api/auth/token/refresh입니다.
TenantJWT tenantId와 X-Tenant-Id가 다르면 403 TENANT_MISMATCH로 차단합니다.
JWT Claimsub, tenantId만 request context에 반영합니다. roles, permissions는 JWT에서 사용하지 않습니다.
JWT SignatureHS256, RS256 서명 검증을 지원합니다. 로컬 기본값은 HS256입니다.
JWT Issuer/Audienceiss는 JWT_ISSUER, aud는 JWT_AUDIENCE와 일치해야 합니다.
Tenant Status기능 활성화 시 tenant status API와 Redis cache로 active tenant만 통과시킵니다.
Security Headersgateway-service가 공통 보안 header를 모든 응답에 추가합니다.
CORS운영 환경에서는 명시된 origin만 허용하고 wildcard origin을 사용하지 않습니다.
TimeoutAuth upstream은 기본 3000ms, Admin/App upstream은 기본 5000ms까지 기다립니다.
RetryGET, HEAD, OPTIONS만 timeout, network error, 502, 503, 504에 대해 기본 1회 재시도합니다.
Rate LimitRedis fixed window 방식으로 route group별 요청 수를 제한합니다.
BodyGET, HEAD를 제외한 요청 body는 upstream으로 전달합니다.
Upstream 응답upstream status, body, header를 전달하되 hop-by-hop header는 제외합니다.

상세

라우팅 맵
/api/auth/**AUTH_IAM_SERVICE_URL로 전달합니다. auth upstream URL은 auth-iam-service 컨트롤러 경로와 맞도록 /api/auth base path까지 포함합니다.
/api/admin/**ADMIN_BFF_SERVICE_URL로 전달합니다. AWS 기본값은 Cloud Map 내부 URL과 admin base path를 포함한 http://admin-bff-service.{env}.multi-tenant.local:3000/api/admin입니다.
/api/app/**USER_BFF_SERVICE_URL로 전달합니다.
Security Headers와 CORS
GATEWAY_SECURITY_HEADERS_ENABLED보안 header 적용 여부입니다. 기본값은 true입니다.
보안 HeaderX-Content-Type-Options, X-Frame-Options, Referrer-Policy, X-Permitted-Cross-Domain-Policies, Permissions-Policy, Content-Security-Policy, Cross-Origin-Opener-Policy, Cross-Origin-Resource-Policy를 적용합니다.
HSTS요청이 HTTPS로 들어온 경우 Strict-Transport-Security를 추가합니다.
GATEWAY_CORS_ALLOWED_ORIGINS쉼표로 구분한 허용 origin 목록입니다. 운영에서는 명시 origin만 허용합니다.
GATEWAY_CORS_ALLOWED_METHODS기본값은 GET,POST,PUT,PATCH,DELETE,OPTIONS입니다.
GATEWAY_CORS_ALLOWED_HEADERSAuthorization, Content-Type, Accept, X-Request-Id, X-Tenant-Id, Idempotency-Key를 허용합니다.
GATEWAY_CORS_EXPOSED_HEADERSX-Request-Id, X-Tenant-Id, rate limit headers를 노출합니다.
Wildcard* origin은 readiness check에서 실패 처리합니다.
비활성화GATEWAY_SECURITY_HEADERS_ENABLED=false이면 gateway-service가 보안 header를 추가하지 않습니다.
Tenant Status Check
GATEWAY_TENANT_STATUS_CHECK_ENABLEDtenant status check 활성화 여부입니다. 코드 기본값은 false이고, 로컬 compose 환경은 실제 tenant-service 연동 검증을 위해 true로 설정합니다.
TENANT_SERVICE_URLtenant 상태를 조회할 내부 서비스 URL입니다.
TENANT_INTERNAL_AUTH_SECRETtenant-service 내부 API 호출에 사용하는 HMAC secret입니다. tenant-service의 값과 같아야 합니다.
조회 EndpointGET /internal/tenants/{tenantId}/status
조회 HeaderAccept: application/json
내부 인증 HeaderX-Internal-Service-Id, X-Internal-Timestamp, X-Internal-Signature를 함께 전송합니다.
Cache keytenant-status:{env}:{tenantId}
Cache TTLGATEWAY_TENANT_STATUS_CACHE_TTL_SECONDS. 기본값은 30초입니다.
TimeoutGATEWAY_TENANT_STATUS_TIMEOUT_MS. 기본값은 1000ms입니다.
통과 상태active
차단 상태provisioning, suspended, deleted는 403 TENANT_NOT_READY로 차단합니다.
예외POST /api/auth/login, POST /api/auth/token/refresh는 tenant status check 대상에서 제외합니다.
Tenant 누락private 업무 API에서 tenant context가 없으면 403 TENANT_REQUIRED로 차단합니다.
조회 실패tenant status 조회 실패는 503 TENANT_STATUS_UNAVAILABLE로 응답합니다.
Redis Rate Limit
저장소로컬 Docker 환경은 compose Redis를 사용하고, AWS 배포 환경은 ElastiCache Redis를 사용합니다.
REDIS_URLgateway-service가 rate limit 저장소로 연결할 Redis URL입니다. ECS 배포에서는 CDK가 ElastiCache endpoint로 주입합니다.
AlgorithmINCR와 EXPIRE를 사용하는 60초 fixed window 방식입니다.
Route group/api/auth/**는 auth, /api/admin/**는 admin, /api/app/**는 app입니다.
인증 후 Keyrate-limit:{env}:gateway:{tenantId}:{userId}:{routeGroup}
인증 전 Keyrate-limit:{env}:gateway:anonymous:{ip}:{routeGroup}
auth기본 60 requests / 60 seconds
admin기본 300 requests / 60 seconds
app기본 600 requests / 60 seconds
초과 응답429 RATE_LIMIT_EXCEEDED
Redis 장애503 RATE_LIMIT_UNAVAILABLE
응답 HeaderX-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset
비활성화GATEWAY_RATE_LIMIT_ENABLED=false이면 rate limit check와 rate limit header를 적용하지 않습니다.
Timeout과 Retry
GATEWAY_AUTH_UPSTREAM_TIMEOUT_MS/api/auth/** upstream timeout입니다. 기본값은 3000입니다.
GATEWAY_ADMIN_UPSTREAM_TIMEOUT_MS/api/admin/** upstream timeout입니다. 기본값은 5000입니다.
GATEWAY_APP_UPSTREAM_TIMEOUT_MS/api/app/** upstream timeout입니다. 기본값은 5000입니다.
GATEWAY_SAFE_METHOD_RETRIESsafe method 재시도 횟수입니다. 기본값은 1입니다.
Retry methodGET, HEAD, OPTIONS만 재시도합니다.
Retry conditionnetwork error, timeout, upstream 502, 503, 504 응답입니다.
Timeout 실패 응답최종 timeout은 504 GATEWAY_TIMEOUT으로 응답합니다.
연결 실패 응답최종 network error는 503 UPSTREAM_UNAVAILABLE로 응답합니다.
쓰기 요청POST, PUT, PATCH, DELETE는 중복 실행 위험 때문에 재시도하지 않습니다.
전파 Header
X-Request-Id요청에 없으면 gateway-service가 생성하고 응답과 upstream 요청에 전파합니다.
X-Tenant-IdJWT tenantId를 기준으로 upstream 요청에 전파합니다.
X-User-IdJWT sub를 기준으로 upstream 요청에 전파합니다.
Upstream 응답 전달
Statusupstream HTTP status를 gateway 응답 status로 그대로 사용합니다.
Bodyupstream response body를 Buffer로 읽어 그대로 반환합니다.
Headerupstream response header를 전달합니다.
제외 Headerconnection, content-length, expect, host, keep-alive, proxy-authenticate, proxy-authorization, te, trailer, transfer-encoding, upgrade는 전달하지 않습니다.
Redirectupstream redirect는 gateway가 자동으로 따라가지 않고 manual 방식으로 처리합니다.
JWT 서명 검증
JWT_ALGORITHM=HS256JWT_SECRET으로 HMAC SHA-256 서명을 검증합니다.
JWT_ALGORITHM=RS256JWT_PUBLIC_KEY로 RSA SHA-256 서명을 검증합니다.
JWT_ISSUER토큰의 iss claim과 일치해야 합니다.
JWT_AUDIENCE토큰의 aud claim과 일치해야 합니다. aud가 배열이면 값 중 하나가 일치해야 합니다.
실패 응답서명, algorithm, issuer, audience가 맞지 않으면 401 AUTH_INVALID_TOKEN으로 응답합니다.
Claim 반영검증 후 sub, tenantId만 request context에 반영합니다.
요청 Body 전달
대상 MethodPOST, PUT, PATCH, DELETE 등 body를 가질 수 있는 요청입니다.
전달 방식gateway-service가 원본 request stream을 upstream으로 전달합니다.
제외 MethodGET, HEAD 요청에는 body를 전달하지 않습니다.
주의gateway-service는 도메인 payload를 해석하지 않습니다.
