gateway-service API

GET /ready

gateway-service가 트래픽을 받을 준비 상태인지 확인합니다. 설정, JWT 검증 준비 상태, proxy route, rate limit, security/CORS 기준을 확인합니다.

GET /ready READY

기본 정보

목적gateway-service가 요청을 받을 준비 상태인지 확인합니다.
인증없음. 운영 readiness check 대상이므로 public endpoint로 둡니다.
현재 범위gateway 설정, JWT verifier 준비 상태, Auth/IAM, Admin BFF, User BFF proxy route URL 형식, upstream timeout/retry 설정, route group별 rate limit 설정, security/CORS 설정을 확인합니다. tenant status check 설정은 각 BFF readiness와 런타임 guard 범위입니다.
외부 연결현재 readiness는 설정 유효성 기준입니다. Redis나 tenant-service에 실제 ping 요청은 보내지 않습니다.
성공 상태200 OK
실패 상태503 Service Unavailable

statusready 또는 not_ready. gateway-service가 요청을 받을 준비 상태인지 나타냅니다.
servicegateway-service. 응답 주체 서비스명입니다.
timestamp응답 생성 시각입니다.
checks.configGATEWAY_PORT, request context header 설정을 확인합니다.
checks.jwtJWT_SECRET/JWT_PUBLIC_KEY, JWT_ISSUER, JWT_AUDIENCE 설정을 확인합니다.
checks.proxyRoutesAuth/IAM, Admin BFF, User BFF upstream URL이 http 또는 https URL인지, timeout/retry 설정이 유효한지 확인합니다.
checks.rateLimitrate limit 활성화 시 REDIS_URL, window seconds, route group별 limit이 유효한지 확인합니다.
checks.securityprod 환경 CORS origin 필수 여부와 wildcard 금지 기준을 확인합니다.

운영 메모
ALB health check현재 ALB target group health check는 /health를 사용합니다.
Readiness 용도/ready는 배포 후 트래픽 수신 가능 여부를 더 엄격하게 판단하는 endpoint입니다.
Rate limitGATEWAY_RATE_LIMIT_ENABLED=true이면 REDIS_URL이 필요합니다.
Tenant statusgateway-service는 tenant-service를 직접 호출하지 않습니다. tenant status check는 Admin BFF와 User BFF가 tenant-service 내부 API로 수행합니다.
Production CORSAPP_ENV=prod에서는 GATEWAY_CORS_ALLOWED_ORIGINS가 비어 있으면 실패합니다.
Wildcard CORS* origin은 모든 환경에서 readiness 실패로 처리합니다.
