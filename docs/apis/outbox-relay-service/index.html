outbox-relay-service API 문서

서비스 DB의 outbox_events를 source별로 relay하고 worker 상태와 source backlog를 확인하는 내부 운영 API입니다.

서비스 상태

상태Relay engine 구현 NestJS shell, source adapter, batch claim, retry/failed 상태 전환, mock/EventBridge/SQS publisher, 감사 이벤트 SQS 기본 발행, 운영 조회 API가 있습니다.
소유 범위source DB의 outbox_events 발행, retry, 실패 기록, 운영 관측
서비스 경계다른 서비스 내부 source code나 generated Prisma client를 import하지 않고 source별 DB URL secret과 SQL adapter로 outbox table만 다룹니다.
감사 이벤트OUTBOX_PUBLISHER_TYPE=sqs이면 CDK 관리형 audit event queue로 발행하고, audit-log-service가 해당 queue를 polling해 저장합니다.

Endpoints

MethodPathCallerStatus설명
GET/healthPublic구현프로세스 생존 확인용 endpoint입니다.
GET/readyPublic구현config, source DB URL, publisher target, worker 설정 기준으로 준비 상태를 확인합니다.
GET/internal/operations/outbox-relay/statusInternal operator구현worker enable 여부, publisher type, 처리 카운터, 마지막 성공/실패 상태를 조회합니다.
GET/internal/operations/outbox-relay/sourcesInternal operator구현source별 pending/failed count와 oldest pending age를 조회합니다.

운영 기준

Worker disabledOUTBOX_WORKER_ENABLED=false이면 source DB URL 없이도 readiness가 통과할 수 있습니다.
Worker enabledOUTBOX_SOURCES에 포함된 source별 DB URL secret이 모두 있어야 readiness가 통과합니다.
Retrypublisher 실패 시 retryCount, lastError, nextRetryAt을 기록하고 최대 재시도 초과 시 failed로 전환합니다.
LocalStack SQS로컬 SQS 검증은 OUTBOX_PUBLISHER_TYPE=sqs, OUTBOX_SQS_QUEUE_URL, OUTBOX_SQS_ENDPOINT=http://localstack:4566을 함께 설정합니다.
SecretDB URL, token, SQL, stack trace는 운영 API 응답이나 lastError에 노출하지 않습니다.
