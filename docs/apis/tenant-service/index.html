tenant-service API 문서

tenant-service는 외부에 직접 노출하지 않는 내부 서비스입니다. 외부 관리자 API는 이후 admin-bff-service의 /api/admin/** surface가 소유하고, tenant-service는 gateway와 BFF가 호출하는 /internal/** 계약을 제공합니다.

API 공통 기준

Public endpointGET /health, GET /ready만 public입니다.
Internal endpoint/internal/**은 HMAC 기반 internal service auth가 필요합니다.
HeadersContent-Type, Accept, X-Request-Id, X-Tenant-Id, X-Internal-Service-Id, X-Internal-Timestamp, X-Internal-Signature
Internal signingMETHOD, originalUrl, timestamp, requestId, body SHA-256 hash를 줄바꿈으로 연결한 문자열에 TENANT_INTERNAL_AUTH_SECRET으로 HMAC-SHA256을 계산합니다. body가 없는 GET 요청은 빈 문자열의 SHA-256 hash를 사용합니다.
Envelope응답은 success, requestId, timestamp, data 또는 error 구조를 사용합니다. meta 필드는 사용하지 않습니다.
Tenant statusprovisioning, active, suspended, deleted를 사용합니다. gateway 업무 API는 active만 통과시킵니다.

Purposegateway-service가 업무 API 요청을 내부 서비스로 넘기기 전에 tenant가 사용 가능한 상태인지 확인합니다.

Errors400 TENANT_INVALID_ID, 404 TENANT_NOT_FOUND, internal auth error codes

POST /internal/admin/tenants

Purposeadmin-bff-service가 외부 관리자 API 요청을 검증한 뒤 tenant 생성을 위임합니다.
Allowed calleradmin-bff-service
Request bodyname은 필수이며, domain은 선택입니다.
Domain입력된 domain은 ASCII hostname으로 정규화해 tenant_domains에 함께 저장합니다. 이미 등록된 domain은 409 TENANT_DOMAIN_CONFLICT로 실패합니다.
Transactiontenant 생성과 tenant.created outbox event 저장을 같은 DB transaction에서 처리합니다.
Success status201 Created
Success datatenantId, code, name, status, dbStrategy, createdAt, updatedAt
Errors400 VALIDATION_FAILED, 400 TENANT_CODE_GENERATION_FAILED, 409 TENANT_DOMAIN_CONFLICT, internal auth error codes
 
{
  "name": "ACME Korea",
  "domain": "factory.acme.co.kr"
}

Tenant Code 규칙

형식tenant.code는 대문자 base 4자리와 sequence 4자리를 붙인 8자리 문자열입니다. 구분자는 사용하지 않습니다.
Domain 우선요청 body에 domain이 있으면 registrable domain label을 base로 사용합니다. acme.example.com은 EXAM0001, factory.acme.co.kr은 ACME0001을 생성합니다.
Name fallbackdomain이 없으면 name을 대문자 ASCII로 정규화합니다. ACME 한국은 ACME0001을 생성합니다.
한글 fallbackASCII label이 없고 한글이 있으면 첫 한글 음절을 로마자로 변환하고 부족한 자리는 hash로 채웁니다. 한국공장은 HANF0001을 생성합니다.
Hash fallbackASCII label도 한글도 없으면 SHA-256 hash의 가운데 4글자를 base로 사용합니다.
Sequence동일 base가 이미 있으면 0002, 0003처럼 sequence를 증가시켜 unique code를 생성합니다.
