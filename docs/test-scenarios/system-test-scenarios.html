System Test Scenarios

전체 시스템 테스트 시나리오

로컬 seed 데이터 준비부터 tenant 생성, 사용자 생성, 권한 관리, WMS 창고/로케이션/자재/재고 업무까지 전체 흐름을 검증하기 위한 시나리오 문서입니다. 1차 테스트는 자동 E2E 코드보다 시나리오 기반 AI 검증을 우선하며, 성공 판정에는 응답 JSON, requestId, DB row, outbox event, audit log 같은 증거를 남깁니다.

문서 기준

대상 환경기본은 local compose와 local DB, LocalStack SQS입니다. dev AWS 검증은 별도 evidence가 준비된 시나리오에서만 수행합니다.
실행 원칙시나리오는 SETUP → TENANT → AUTH → ACCESS → WMS → OBS 순서로 실행합니다.
공통 HeadersAuthorization, Content-Type, Accept, X-Request-Id, X-Tenant-Id, mutation에는 Idempotency-Key를 사용합니다.
응답 Envelope모든 API 응답은 success, requestId, timestamp를 포함하고 성공은 data, 실패는 error.code를 포함해야 합니다.
상태 표시Implemented는 현재 API/스크립트가 있는 항목, Manual은 기능은 있으나 자동 시나리오에 아직 묶지 않은 항목, Planned는 화면 또는 후속 기능 구현 후 실행할 항목입니다.

NEG-IDEMP-001 mutation idempotency key 검증 P0 중복 실행 위험이 있는 POST/PATCH/PUT API가 있습니다. Idempotency-Key 없이 호출하고, 같은 key로 같은 요청을 반복합니다. key 누락은 IDEMPOTENCY_KEY_REQUIRED로 실패하고 같은 key 반복은 중복 row를 만들지 않습니다. error response, duplicate count, idempotency key Implemented
NEG-RATE-001 gateway rate limit P1 rate limit 설정이 활성화되어 있습니다. 같은 client 식별자로 제한보다 많은 요청을 짧은 시간에 보냅니다. 429와 rate limit error code가 반환되고 정상 요청 로그와 구분됩니다. response headers, gateway log Manual
