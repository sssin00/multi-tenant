Performance Test Criteria

성능 테스트 기준

멀티테넌트 스마트 팩토리 1차 WMS 기준 성능 목표입니다. 평균 응답 시간이 아니라 p95/p99, 오류율, 자원 사용률, 이벤트 지연, 데이터 정합성을 함께 합격 조건으로 봅니다.

기본 원칙

기준일2026-06-23
대상gateway, Admin BFF, User BFF, Auth/IAM, Tenant, WMS, Audit Log, Outbox Relay, web-admin, web-wms
측정 위치외부 사용자의 요청은 gateway-service 진입점 기준으로 측정합니다. domain/internal service는 병목 분석용으로만 직접 측정합니다.
합격 기준p95, p99, 오류율, 5xx, timeout, outbox/audit 지연, DB 정합성 조건을 모두 만족해야 통과입니다.
평균값 사용평균 응답 시간은 참고값입니다. 합격/불합격은 p95/p99와 오류율로 판단합니다.
테넌트 격리부하 중에도 tenant mismatch, cross-tenant 데이터 노출, 권한 우회, audit/outbox tenantId 누락은 0건이어야 합니다.
운영 리소스운영 AWS/ERP 리소스는 성능 테스트에서 직접 호출하지 않습니다. local, mock, dev 환경만 사용합니다.

업무 Mix

읽기/쓰기 비율WMS 일반 부하는 조회 85%, 변경 15%로 시작합니다. 피킹/출하 집중 시간은 조회 70%, 변경 30%를 별도 측정합니다.
사용자 비율일반 WMS 사용자 80%, tenant 관리자 15%, system admin 5% 이하로 구성합니다.
조회 분포dashboard 15%, 현재고 25%, 스냅샷 10%, 기준정보 15%, 출고/포장 목록 25%, app shell/navigation 10%
변경 분포재고 조정 15%, 입고 확정 20%, 출고 할당 25%, 포장 25%, 출하 15%
Think time화면 기반 시나리오는 요청 사이 1s~5s 랜덤 think time을 둡니다. API 한계 측정은 별도 stress 프로파일에서만 think time을 줄입니다.

불합격 기준

즉시 실패

5xx 1건 이상, 정합성 오류 1건 이상, tenant 교차 데이터 노출, 권한 우회, outbox/audit 누락이 발견되면 응답 시간이 좋아도 실패입니다.

조건부 실패

p95 또는 p99 기준 초과, timeout 발생, outbox backlog 회복 실패, CPU/메모리/DB connection 기준 초과는 병목 분석 후 재측정해야 합니다.
