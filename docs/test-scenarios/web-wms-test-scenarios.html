Web WMS Test Scenarios

web-wms 테스트 시나리오

사용자용 WMS Desktop 화면의 로그인, 세션 초기화, 권한 기반 메뉴, 대시보드, 재고, 스냅샷, 기준정보, 출고 조회, 재고/입고/출고 mutation, 감사 로그, 오류 처리를 시나리오 기반으로 검증합니다. 저장소에는 별도 web-user 앱이 없으므로 사용자 화면 검증 대상은 apps/web-wms와 user-bff-service입니다.

공통 실행 기준

대상 앱apps/web-wms. Docker 로컬과 Vite 단독 개발 서버 모두 web-wms 기준 URL은 http://localhost:5174/입니다.
API 진입점gateway http://localhost:3000 경유 /api/auth/**, /api/app/**
Seed 계정admin@demo.local, tenant 11111111-1111-4111-8111-111111111111. 비밀번호와 token 원문은 evidence에 남기지 않습니다.
공통 HeadersAuthorization, X-Tenant-Id, X-Request-Id
성공 Evidence화면 상태 설명, API response JSON 요약, requestId, list count, audit log row, API 데이터 또는 샘플 데이터 표시
실패 Evidence실제 결과, 기대 결과, 재현 단계, error.code, HTTP status, requestId
