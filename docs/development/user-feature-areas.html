User Feature Areas

사용자별 기능 영역 정의서

현재 구현된 인증, BFF, WMS, 감사/이벤트 흐름을 기준으로 사용자 타입과 역할별 기능 영역을 정리합니다. 이 문서는 화면 메뉴와 권한 설계, 테스트 시나리오, 향후 web-admin/web-wms 구현의 기준으로 사용합니다.

기준

기준일2026-06-21
사용자 타입코드상 사용자 타입은 system_admin과 general_user입니다.
Tenant 역할general_user에게 tenant role을 부여해 tenant_admin, tenant_user, WMS 담당자 역할을 구성합니다.
중요 경계system_admin은 tenant-scoped 사용자 API로 생성하거나 tenant role에 섞지 않습니다. 현재 구현은 이 분리를 검증하고, 전용 system admin bootstrap/API는 후속 범위입니다.
검증 증거frontend-wms-api-verification.html에서 33개 시나리오 통과를 확인했습니다.

사용자 분류

분류 식별 기준 주요 목적 현재 구현 상태
system_admin 사용자 타입 system_admin 플랫폼 전체 tenant 운영, bootstrap, 시스템 관리 정책을 담당하는 별도 영역입니다. tenant-scoped API에서 생성/role mapping을 차단합니다. 전용 bootstrap과 운영 API는 후속입니다.
tenant_admin general_user + tenant 전체 scope role 자기 tenant의 사용자, role, permission, tenant module/domain, audit 조회, WMS 운영 권한을 관리합니다. Admin BFF와 Auth/IAM permission check로 구현되어 있습니다. warehouse scope assignment는 차단합니다.
tenant_user general_user + 업무 role WMS app shell, navigation, WMS 조회/작업 API를 부여받은 권한 범위 안에서 사용합니다. User BFF app shell/navigation과 WMS 화면 조회 API가 구현되어 있습니다. 재고 조회 화면은 wms.inventory.read, 창고/자재 관리 화면은 관리 권한에 따라 노출됩니다.
WMS 담당자 general_user + WMS permission set 창고/로케이션/품목, 재고, 입고, 출고, 포장, 출하, 스냅샷 업무를 수행합니다. 권한별 WMS API가 구현되어 있으며, lot/serial과 작업 지시는 후속입니다.

구현과 후속 범위

구현 완료

tenant/user/role/permission 관리, tenant admin과 tenant user 범위 분리, User BFF app shell/navigation, WMS 기준정보/재고/스냅샷/입고/출고/포장/출하, outbox/audit 검증.

후속

system admin 전용 bootstrap/API, tenant proxy 접근 정책, User BFF WMS 화면 조합 API, web-admin/web-wms 화면, lot/serial, 작업 지시, ERP retry.
