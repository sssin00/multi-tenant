web-admin 개발과정

운영자와 관리자를 위한 React + Vite 프론트엔드 앱입니다.

서비스/앱 역할

위치apps/web-admin
카테고리Frontend App · Admin UI
대상시스템 운영자, 테넌트 관리자
연결admin-bff-service를 통해 관리 화면 데이터를 조회합니다.
초기 화면테넌트 관리, 사용자/권한 관리, 운영 설정, 감사 로그

API 문서화면 연동 기준은 docs/apis/admin-bff-service/index.html를 따릅니다.

기능별 작업 내역

Tenant Admintenant 목록, 상세, 생성, 기본 정보 수정, 상태 변경, module/domain 관리를 Admin BFF API와 연결합니다.
User Admin사용자 목록/상세/생성/상태 변경과 role 부여/해제를 Auth/IAM 경계에 맞춰 구현합니다.
Access Controlrole/permission 화면은 BFF의 screen-data 조합 API를 사용하고, 변경성 요청은 idempotency 기준을 따릅니다.
Auditaudit-log-service 저장소 연결 후 검색 조건, paging, 상세 evidence 표시를 구현합니다.

테스트 시나리오

WEB-ADMIN-001권한이 있는 관리자는 tenant 목록과 상세 화면을 조회할 수 있다.
WEB-ADMIN-002권한이 없는 메뉴와 변경 버튼은 표시되지 않거나 API 호출 전 차단된다.
WEB-ADMIN-003tenant 상태/module/domain 변경 후 목록과 상세 화면이 갱신된다.

후속 작업

현재 web-admin 착수 전 이어서 확정할 작업입니다. 이 표의 작업이 진행되면 관련 코드와 문서를 함께 갱신합니다.

우선순위 작업 범위 완료 기준 함께 갱신할 문서
P0 Admin session 초기화 화면 Admin BFF의 /api/admin/me 또는 permission summary 응답을 기준으로 메뉴와 접근 가능 화면을 렌더링합니다. 로그인 후 관리자 권한과 tenant scope에 따라 접근 가능한 메뉴만 표시됩니다. admin-bff-service 개발 문서, web-admin 개발 문서, Auth/IAM 개발 문서
P0 운영 UX 기준 tenant 생성, 상태 변경, module 교체, domain 비활성화 같은 위험 작업에 확인 dialog와 결과 evidence 표시를 추가합니다. 위험 작업 수행 전 사용자 확인을 받고 완료 후 requestId와 변경 결과를 화면에서 확인할 수 있습니다. web-admin 개발 문서, admin-bff-service API 문서, testing 규칙
P1 Scenario Evidence 절차 AI 검증 시 화면 상태, API 응답, requestId를 함께 남길 수 있는 체크 절차를 정리합니다. 테스트 시나리오별 evidence가 화면 캡처, 응답 JSON, requestId 기준으로 재현 가능합니다. web-admin 개발 문서, testing 규칙, 개발 로그
