Development Process

wms-service 개발과정

WMS 기준정보, 재고, 입고, 출고, 작업 지시를 담당하는 첫 메인 도메인 서비스입니다.

서비스 역할

위치apps/wms-service
초기 구조하나의 서비스 내부를 모듈형 모놀리스로 구성
도메인 순서warehouse → location → item → inventory → inbound → outbound → task → shipment
이벤트wms.inventory.adjusted, wms.inbound.confirmed, wms.outbound.allocated, wms.outbound.shipped

첫 구현 범위

기준정보

창고, 로케이션, 품목, 거래처, 단위, 바코드

재고

재고 조회, 이동, 조정, 로트/시리얼 정책

입출고

입고 예정/검수/적치, 출고 요청/할당/피킹

개발 기록

아직 구현 전입니다. WMS 모듈이 생길 때마다 API, DB, 이벤트, 테스트를 이곳에 기록합니다.
