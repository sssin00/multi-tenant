Development Process

user-bff-service 개발과정

WMS 사용자 화면과 PDA Web에 필요한 데이터를 조합하는 사용자용 BFF입니다.

서비스 역할

위치apps/user-bff-service
책임사용자 정보, 현재 테넌트, 사용 가능 메뉴, WMS 화면 초기 데이터, 대시보드 응답 조합
제작 시점Gateway, Tenant, Auth/IAM 최소 구현 후 WMS 화면 개발 직전에 시작
주의재고 차감, 입고 완료, 출고 할당 정책 같은 도메인 업무 로직은 WMS에 둡니다.

첫 구현 범위

GET /me

사용자, tenant, roles, permissions summary

GET /app-shell

메뉴, 테넌트 설정, 사용 가능 모듈

GET /wms/summary

창고 수, 품목 수, 재고 요약, 입출고 대기 건수

개발 기록

아직 구현 전입니다. 화면별 API가 생길 때마다 이 문서에 기록합니다.
