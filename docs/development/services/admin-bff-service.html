Development Process

admin-bff-service 개발과정

운영자/관리자 화면에 필요한 테넌트, 사용자, 권한, 설정, 감사 로그 API를 조합합니다.

서비스 역할

위치apps/admin-bff-service
책임테넌트 관리, 사용자/권한 관리, 운영 설정, 감사 로그 조회 화면 API 조합
제작 시점초기 운영자 화면이 필요할 때 최소 기능부터 제작
주의권한의 최종 판단은 Auth/IAM에 두고, 업무 정책은 각 도메인 서비스에 둡니다.

첫 구현 범위

Tenants

테넌트 목록, 생성, 상태 변경 화면 API

Users

사용자 목록, 초대, 역할 부여 화면 API

Audit

운영자용 감사 로그 조회 API
