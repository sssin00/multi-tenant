web-wms 개발과정

WMS 사용자와 현장 작업자를 위한 React + Vite 프론트엔드 앱입니다.

서비스/앱 역할

위치apps/web-wms
카테고리Frontend App · WMS UI
대상창고 운영자, 현장 작업자, PDA Web 사용자
연결user-bff-service를 통해 WMS 화면 데이터를 조회합니다.
초기 화면대시보드, 기준정보, 재고, 입고, 출고

구현 상태

상태착수 전
선행 조건wms-service의 1차 조회 API와 user-bff-service 화면 조합 API가 준비되어야 합니다.
API 문서사용자 화면 API 계약은 이후 docs/apis/user-bff-service/index.html에 기록합니다.

남은 작업

기능별 작업 내역

App Shell사용자, tenant, 활성 module, 권한 summary를 기반으로 navigation과 route guard를 구성합니다.
Inventory재고 검색, lot/serial 표시, 조정 요청 흐름을 WMS API 계약에 맞춰 구현합니다.
Inbound입고 예정, 검수, 적치 상태를 현장 작업자가 빠르게 확인할 수 있게 구성합니다.
Outbound출고 요청, 할당, 피킹, 출하 상태를 작업 중심 화면으로 정리합니다.

테스트 시나리오

WEB-WMS-001활성 tenant와 WMS module 권한이 있는 사용자는 WMS navigation을 볼 수 있다.
WEB-WMS-002재고 목록 검색 조건과 paging이 API 응답과 화면 상태에 일치한다.
WEB-WMS-003입고/출고 작업 상태 변경 후 목록과 상세 화면이 같은 상태를 표시한다.

후속 작업

현재 web-wms 착수 전 이어서 확정할 작업입니다. 이 표의 작업이 진행되면 관련 코드와 문서를 함께 갱신합니다.

우선순위 작업 범위 완료 기준 함께 갱신할 문서
P0 WMS 첫 화면 연동 User BFF의 screen-data 응답을 기준으로 dashboard, inventory summary, 작업 목록 화면을 구성합니다. tenant와 permission summary가 반영된 첫 화면이 실제 API 응답으로 렌더링됩니다. user-bff-service 개발 문서, web-wms 개발 문서, wms-service API 문서
P1 PDA Web UX 스캔 입력, 큰 터치 target, 네트워크 불안정 상황의 재시도 UX를 별도로 검증합니다. 현장 장비 화면에서 주요 버튼과 입력이 안정적으로 조작되고 실패 재시도가 명확히 표시됩니다. web-wms 개발 문서, testing 규칙, WMS 시나리오 문서
P1 현장 검증 evidence WMS 핵심 흐름은 실제 창고 작업 순서에 맞춰 시나리오 기반 AI 검증 evidence를 남깁니다. 입고, 재고 조정, 출고 흐름별 화면 상태, API 응답, requestId, audit/outbox evidence가 연결됩니다. web-wms 개발 문서, wms-service 개발 문서, testing 규칙

  • 입고, 재고, 출고 화면은 tenant context, 권한 summary, 현장 작업 evidence를 함께 확인하는 기준으로 유지합니다.
