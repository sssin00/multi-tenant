Development Process

outbox-relay-service 개발과정

서비스 DB의 Outbox 이벤트를 읽어 EventBridge/SQS로 안정적으로 발행합니다.

서비스 역할

위치apps/outbox-relay-service
책임미발행 outbox event 조회, publish, retry, 실패 기록
초기 대상WMS 재고 변경, 입고 완료, 출고 완료 이벤트
주의도메인 트랜잭션은 각 서비스 DB에서 끝나고, relay는 발행만 담당합니다.

첫 구현 범위

Polling

미발행 이벤트 조회와 lock 전략

Publish

EventBridge event bus로 발행

Retry

실패 횟수, DLQ 연계, 수동 replay 기준
