프로젝트 공통 규칙

공통 규칙은 한 페이지에 모두 넣지 않고 카테고리별 문서로 분리합니다. 이 페이지는 전체 규칙의 목차와 진행 상태를 확인하는 허브입니다.

공통 규칙 상태 목록

프로젝트 규칙은 확정된 규칙, 초안 확정 규칙, 결정 필요 규칙으로 나눠 관리합니다.

결정 필요 규칙

다음 규칙 결정 순서

1DB 규칙 - WMS 1차 schema는 구현됐지만 table, column, migration, index, audit field 공통 기준은 별도로 정합니다.
2WMS 2차 도메인 규칙 - lot/serial, 보류 재고, 작업 지시, ERP 재전송 기준을 정합니다.
3Idempotency 규칙 - 변경성 POST/PATCH 요청의 중복 처리 저장소와 TTL을 정합니다.
4Audit-log-service 규칙 - 보존 기간, archive, replay 감사 기록, 운영 장애 처리 기준을 정합니다.
5Outbox Relay 규칙 - replay 승인, custom metric/alarm, dev AWS end-to-end 운영 기준을 정합니다.
6운영 JWT 규칙 - RS256, key rotation, JWKS 또는 public key 배포 방식을 정합니다.
7Git/PR 운영 규칙 - 브랜치 보호, 리뷰, merge 전략, PR 크기 기준을 정합니다.
8배포/운영 상세 규칙 - rollback, alarm, runbook, 비용 절감용 desired count 운영을 정합니다.
9파일/S3 규칙 - 업로드 검증, object key, presigned URL 만료, 보존 정책을 정합니다.
10ERP 연동 규칙 - retry, DLQ, 수동 재전송, 외부 장애 시 업무 상태를 정합니다.
