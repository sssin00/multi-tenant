시스템 로그와 감사 로그 규칙

시스템 로그, 감사 로그, 로그 보안 기준을 분리해 정리합니다.

시스템 로그와 감사 로그 분리

시스템 로그는 장애 분석과 운영 모니터링을 위한 기록이고, 감사 로그는 사용자와 관리자의 업무 행위를 증명하기 위한 기록입니다. 두 로그는 목적, 저장소, 보존 정책을 분리합니다.

시스템 로그요청, 에러, 성능, 외부 연동, 내부 처리 상태를 기록합니다. 기본 저장소는 CloudWatch Logs입니다.
감사 로그누가, 언제, 어떤 리소스에 어떤 행위를 했는지 기록합니다. 별도 audit 저장소에 append-only로 저장합니다.
분리 기준장애 원인을 찾기 위한 기록이면 시스템 로그, 행위 책임을 증명해야 하는 기록이면 감사 로그입니다.
연결 기준시스템 로그와 감사 로그는 동일한 requestId로 연결할 수 있어야 합니다.

시스템 로그 규칙

시스템 로그는 JSON 구조화 로그로 남기고, 서비스 운영과 장애 분석에 필요한 정보만 포함합니다.

공통 필드timestamp, level, service, env, requestId, tenantId, userId, message
요청 로그method, path, statusCode, durationMs를 포함합니다.
에러 로그errorCode, operation, resourceType, resourceId를 가능한 범위에서 포함합니다.
TraceOpenTelemetry 또는 X-Ray 사용 시 traceId, spanId를 포함합니다.
저장소초기에는 서비스별 CloudWatch Logs log group으로 분리합니다.
 
{
  "timestamp": "2026-06-02T01:31:00.000Z",
  "level": "info",
  "service": "gateway-service",
  "env": "dev",
  "requestId": "req_01HZABC123",
  "tenantId": "tenant_001",
  "userId": "user_001",
  "method": "POST",
  "path": "/api/app/wms/outbound-orders",
  "statusCode": 201,
  "durationMs": 82,
  "message": "Request completed"
}

감사 로그 규칙

감사 로그는 업무 행위 추적용입니다. 수정과 삭제를 허용하지 않는 append-only 원칙을 따르고, 관리자 화면에서 tenant, actor, action, 기간 기준으로 조회할 수 있게 설계합니다.

공통 필드auditId, occurredAt, tenantId, actor, action, resource, result, requestId
선택 필드reason, changes, sourceIp, userAgent를 필요한 경우 포함합니다.
기록 대상권한 변경, 테넌트 설정 변경, 사용자 변경, 재고 조정, 입고 확정, 출고 할당, 출고 확정, ERP 수동 재시도 같은 업무 행위입니다.
기본 제외목록 조회, 단순 상세 조회, 검색, 화면 진입, health check, 내부 polling은 기본적으로 감사 로그 대상이 아닙니다.
민감 조회개인정보 상세 조회, 권한 목록 조회, 시스템 관리자 화면 접근, VIP 테넌트 데이터 접근은 감사 로그 대상으로 둘 수 있습니다.
초기 구현감사 로그 전담 서비스명은 audit-log-service로 확정했습니다. 현재는 구현 전이며, Auth/IAM과 Admin BFF는 AUDIT_LOG_SERVICE_URL이 설정되면 내부 API로 저장/조회 요청을 보내는 방향입니다.
 
{
  "auditId": "aud_01HZABC123",
  "occurredAt": "2026-06-02T01:35:00.000Z",
  "tenantId": "tenant_001",
  "actor": {
    "userId": "user_001",
    "type": "user"
  },
  "action": "wms.outbound.ship.confirmed",
  "resource": {
    "type": "outboundOrder",
    "id": "out_001"
  },
  "result": "success",
  "requestId": "req_01HZABC123",
  "reason": "출고 확정",
  "changes": {
    "before": { "status": "allocated" },
    "after": { "status": "shipped" }
  }
}

로그 보안 규칙

금지password, accessToken, refreshToken, Authorization header, cookie, secret, apiKey, privateKey는 남기지 않습니다.
개인정보전화번호, 이메일, 사업자등록번호 같은 개인정보와 식별정보는 원문 저장하지 않고 필요한 경우 마스킹합니다.
Stack TraceAPI 응답에는 stack trace를 노출하지 않습니다. 운영 로그에서도 필요한 경우에만 제한적으로 남깁니다.
업무 상태 변경중요한 업무 상태 변경은 시스템 로그만으로 대체하지 않고 감사 로그 또는 도메인 이벤트로 별도 기록합니다.
 
authorization: [REDACTED]
email: s***@example.com
phone: 010-****-1234
