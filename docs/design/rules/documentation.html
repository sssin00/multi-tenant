변경 유형별로 어떤 문서를 갱신해야 하는지 정리합니다.

문서 업데이트 규칙

코드, 설계, 운영 기준이 바뀌면 관련 문서를 함께 갱신합니다. 문서 업데이트 규칙의 목적은 설계 문서와 실제 구현이 달라지는 것을 줄이고, 다른 컴퓨터나 다른 작업자가 이어서 볼 수 있게 하는 것입니다.

설계 변경docs/design/* 문서를 수정합니다.
서비스 구현 변경해당 서비스 문서 docs/development/services/{service}.html을 수정합니다.
전체 진행 상태 변경docs/index.html의 요약과 상태를 수정합니다.
Codex 작업 기준 변경Codex가 계속 따라야 하는 규칙이면 AGENTS.md도 수정합니다.
아키텍처 구조 변경관련 설계 문서와 docs/design/smart-factory-aws-ecs.drawio를 함께 수정합니다.
인프라 현황 변경docs/infra/YYYY-MM-DD/index.html에 날짜별 snapshot으로 기록합니다. 날짜가 바뀌면 새 폴더를 만들고, 같은 날짜에 여러 번 바뀌면 해당 날짜의 기존 HTML을 수정합니다.
API 변경API를 만들거나 수정하면 docs/apis/{service}/, 해당 서비스 개발 문서, postman/multi-tenant-local.postman_collection.json을 함께 갱신합니다.

문서별 책임

docs/index.html전체 문서 대시보드, 현재 진행 상태, 주요 결정 요약, 문서 링크를 관리합니다.
docs/design/project-rules.html네이밍, API, 인증/인가, 멀티테넌트, 에러, 로그, 이벤트, 환경 변수, 테스트 같은 공통 규칙을 관리합니다.
docs/design/aws-ecs-tech-stack.htmlAWS, ECS, VPC, 기술스택, 운영 인프라 결정을 관리합니다.
docs/design/msa-wms-roadmap.htmlWMS first 개발 순서와 단계별 로드맵을 관리합니다.
docs/design/smart-factory-aws-ecs.drawioAWS ECS 기반 스마트 팩토리 아키텍처 설계도를 관리합니다.
docs/infra/YYYY-MM-DD/index.html날짜별 인프라 현황 snapshot을 관리합니다. 날짜가 바뀌면 새 날짜 폴더를 만들고, 같은 날짜의 반복 변경은 같은 파일에 누적합니다.
docs/development/services/{service}.html서비스별 개발 과정, 구현 기능, API, 테스트 시나리오, 이슈, 결정사항을 관리합니다.
postman/multi-tenant-local.postman_collection.json로컬 API 호출 예시와 검증용 request collection을 관리합니다. API 추가/수정/삭제 시 함께 갱신합니다.
AGENTS.mdCodex가 작업할 때 따라야 하는 실행 규칙을 관리합니다.

변경 유형별 문서 업데이트

API 추가/수정/삭제docs/apis/{service}/, 해당 서비스 개발 문서, postman/multi-tenant-local.postman_collection.json을 함께 수정합니다.
WMS API 추가docs/apis/wms-service/, docs/development/services/wms-service.html, Postman collection, 필요 시 테스트 시나리오와 API 규칙을 수정합니다.
공통 규칙 추가docs/design/project-rules.html을 수정하고, Codex가 따라야 하면 AGENTS.md도 수정합니다.
AWS 구성 변경docs/design/aws-ecs-tech-stack.html과 필요한 경우 draw.io 설계도를 수정합니다.
인프라 현황 변경오늘 날짜의 docs/infra/YYYY-MM-DD/index.html을 수정합니다. 오늘 날짜 폴더가 없으면 새로 만들고, docs/index.html의 아키텍처 현황 목록에 링크를 추가합니다.
서비스 추가docs/index.html에 링크를 추가하고, docs/development/services/{service}.html을 생성합니다.
이벤트 추가docs/design/project-rules.html의 이벤트 규칙 또는 해당 서비스 개발 문서에 이벤트 이름과 payload를 기록합니다.
테스트 시나리오 추가해당 서비스 개발 문서에 scenarioId, 사전 조건, 실행 단계, 기대 결과, 증거 기준을 기록합니다.
