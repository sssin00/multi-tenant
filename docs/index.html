개발과정

개발이 시작되면 각 서비스 문서에 구현 내용, 결정사항, API, 테스트, 이슈를 계속 추가합니다.

SERVICES

gateway-service

1차 구현

요청 컨텍스트, tenant 식별, JWT 검증, 라우팅, rate limit

Docker, health/ready, proxy, Redis rate limit, CDK 배포 구성이 정리되었습니다. tenant status check는 BFF가 담당합니다.

admin-bff-service

1차 구현

WMS 전에 tenant, 사용자, role, module 운영 기반 API 조합

Tenant 관리, 사용자/role/permission 화면 API, audit log 조회 route/client, CDK 배포 준비까지 진행되었습니다.

user-bff-service

대기

WMS 1차 API 이후 사용자 화면 데이터와 app shell API 조합

WMS 조회 API와 web-wms 첫 화면 데이터 형태가 정리된 뒤 착수합니다.

tenant-service

1차 구현

테넌트, 모듈, 도메인, DB 전략 관리

Tenant status, admin tenant 관리, module/domain 내부 API, seed 데이터, CDK 배포 준비가 정리되었습니다.

auth-iam-service

1차 구현

로그인, JWT, 사용자, 역할, 권한, tenant scope

Login/refresh/logout, user/role/permission, permission summary/check, audit/outbox, seed 데이터가 구현되었습니다.

wms-service

착수 전

기준정보, 재고, 입고, 출고, 작업 지시

입고, 재고 조정, 출고 할당/확정, audit/outbox 저장을 첫 구현 범위로 유지합니다.

outbox-relay-service

착수 전

Outbox 이벤트 발행, EventBridge/SQS 연동, retry

Auth/IAM과 tenant-service의 outbox_events relay 대상, retry, DLQ 정책 확정이 선행 작업입니다.

audit-log-service

착수 전

감사 로그 저장, 조회, 보존, evidence 관리

Auth/IAM과 Admin BFF의 audit 연동 대상이며 append-only 감사 로그 저장소를 담당합니다.

web-wms

착수 전

WMS 사용자와 현장 작업자용 React + Vite 앱

WMS 1차 API와 User BFF screen-data 계약 이후 화면 구현을 시작합니다.

web-admin

착수 전

운영자와 관리자를 위한 React + Vite 앱

Admin BFF tenant/user/role/permission API를 기준으로 첫 운영 화면을 연결합니다.
