개발과정

개발이 시작되면 각 서비스 문서에 구현 내용, 결정사항, API, 테스트, 이슈를 계속 추가합니다.

SERVICES

gateway-service

요청 컨텍스트, tenant 식별, JWT 검증, 라우팅, rate limit

user-bff-service

WMS 사용자 화면 데이터 조합과 app shell API

admin-bff-service

운영자 화면, 테넌트/사용자/권한/감사 로그 API 조합

tenant-service

테넌트, 모듈, 도메인, DB 전략 관리

auth-iam-service

로그인, JWT, 사용자, 역할, 권한, tenant scope

wms-service

기준정보, 재고, 입고, 출고, 작업 지시

outbox-relay-service

Outbox 이벤트 발행, EventBridge/SQS 연동, retry

web-wms

WMS 사용자와 현장 작업자용 React + Vite 앱

web-admin

운영자와 관리자를 위한 React + Vite 앱
