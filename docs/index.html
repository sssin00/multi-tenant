gateway-service

1차 구현

요청 컨텍스트, tenant 식별, JWT 검증, 라우팅, rate limit

Docker, health/ready, proxy, Redis rate limit, CDK 배포 구성이 정리되었습니다. tenant status check는 BFF가 담당합니다.

admin-bff-service

1차 구현

WMS 전에 tenant, 사용자, role, module 운영 기반 API 조합

Tenant 관리, 사용자/role/permission 화면 API, audit log 조회 route/client, CDK 배포 준비까지 진행되었습니다.

user-bff-service

WMS 화면 API

사용자 화면 데이터, app shell API, WMS 화면 API 조합

NestJS 기본 구조, /api/app/me, /api/app/navigation, /api/app/wms 화면 조회 API, gateway /api/app route, audit-log best-effort 기록, CDK/ECS/workflow가 준비되었습니다.

tenant-service

1차 구현

테넌트, 모듈, 도메인, DB 전략 관리

Tenant status, admin tenant 관리, module/domain 내부 API, seed 데이터, CDK 배포 준비가 정리되었습니다.

auth-iam-service

1차 구현

로그인, JWT, 사용자, 역할, 권한, tenant scope

Login/refresh/logout, user/role/permission, userType 분리, permission summary/check, audit/outbox, seed 데이터가 구현되었습니다.

wms-service

1차 구현 · CDK

기준정보, 재고, 스냅샷, 입고, 출고, 포장, 출하

기준정보, 재고 조정, 일 단위 스냅샷, 입고 확정, 출고 할당/포장/출하, outbox/audit 연계와 CDK/ECS/workflow 기준을 반영했습니다.

outbox-relay-service

1차 구현

Outbox 이벤트 발행, 감사 SQS 전달, retry

Auth/IAM, tenant-service, wms-service outbox relay, mock/EventBridge/SQS publisher, audit event queue/DLQ, Docker/CDK/GitHub Actions 배포 준비가 정리되었습니다.

audit-log-service

1차 구현

감사 로그 저장, 조회, 보존, evidence 관리

Append-only 저장/조회 API, SQS outbox 이벤트 소비, Admin BFF 조회 연동, Docker/CDK/ECS 배포 준비가 정리되었습니다.

web-wms

착수 전

WMS 사용자와 현장 작업자용 React + Vite 앱

User BFF의 /api/app/wms/dashboard, warehouses, materials, inventory-summary 계약을 기준으로 화면 구현을 시작합니다.

web-admin

착수 전

운영자와 관리자를 위한 React + Vite 앱

Admin BFF tenant/user/role/permission API를 기준으로 첫 운영 화면을 연결합니다.
