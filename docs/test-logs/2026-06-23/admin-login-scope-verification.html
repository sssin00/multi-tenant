구현 현황

영역상태내용
Auth/IAM login완료tenant 없는 관리자 로그인에서 email/password로 관리자 후보를 조회하고, 정확히 하나의 관리자 계정만 일치할 때 token을 발급합니다.
중복 관리자 후보완료동일 email/password에 여러 관리자 계정이 일치하면 409 AUTH_LOGIN_AMBIGUOUS로 실패해 범위를 임의 선택하지 않습니다.
web-admin완료로그인 화면에서 테넌트 ID 입력을 제거했고, 로그인 요청에 body tenantId와 X-Tenant-Id header를 보내지 않습니다.
web-wms/web-user유지일반 사용자 로그인은 기존처럼 tenant ID 입력과 tenant header/body 전송을 유지합니다.
문서/Postman완료Auth/IAM API 문서, 서비스 개발 문서, web-admin 시나리오, Postman gateway 관리자 로그인 요청을 현재 정책으로 갱신했습니다.

주의 사항

tenantId 제거 범위테넌트 ID는 시스템에서 제거하지 않았습니다. 제거 대상은 web-admin 로그인 화면의 입력과 로그인 요청의 tenant 값뿐입니다.
로그인 후 scope로그인 이후 API 요청은 JWT의 tenantId를 기준으로 기존 tenant isolation 규칙을 적용합니다.
일반 사용자일반 사용자 로그인은 계속 tenant ID를 함께 받습니다. web-wms 로그인 화면과 API client는 이 동작을 유지합니다.
