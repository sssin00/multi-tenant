발견 사항

새 outbox envelope는 정상

현재 auth-iam-service와 tenant-service의 OutboxEventService는 payload에 aggregateType과 aggregateId를 포함합니다. 이 형식의 이벤트는 relay 검증을 통과했습니다.

오래된 pending event는 실패 처리

기존 로컬 DB에 남아 있던 이벤트 중 payload에 aggregate 필드가 빠진 row는 relay가 계속 재시도한 뒤 failed로 전환했습니다. 이는 실패 격리 동작이 정상임을 보여줍니다.

SQS 연동 통과

OUTBOX_SQS_ENDPOINT를 추가하고 일반 queue에서는 FIFO 전용 옵션을 보내지 않도록 수정한 뒤 outbox-relay → SQS → audit-log-service 저장을 확인했습니다.

Audit Prisma 정합성 수정

꼬인 generated Prisma 폴더를 재생성하고 로컬 audit_log DB에 최신 schema를 push해 event_id column과 index를 맞췄습니다.
