Verification · 2026-06-22

로컬 Gateway 단일 진입점과 Web Docker 검증

로컬 Docker Compose에서도 Admin/User BFF host port를 열지 않고, web-admin/web-wms가 gateway만 API 진입점으로 사용하도록 변경한 결과입니다.

검증 요약

대상docker/local/docker-compose.yml, docker/services/Dockerfile.web, CDK synth 산출물, Postman collection, 시스템 테스트 시나리오
결과통과 BFF host port 미노출, web-admin/web-wms Docker image build, nginx static smoke, CDK gateway-only ALB/BFF ingress 확인
업무 API 진입점http://localhost:3000 gateway
Web URLshttp://localhost:5173 web-admin, http://localhost:5174 web-wms

Evidence

항목명령확인 결과
BFF host port 제거 docker compose -f docker/local/docker-compose.yml config | rg 'published: "3003"|published: "3004"' 매칭 없음. Admin/User BFF는 expose: 3000만 사용합니다.
Web Docker build docker compose -f docker/local/docker-compose.yml build web-admin web-wms multi-tenant-local-web-admin:latest, multi-tenant-local-web-wms:latest 이미지 빌드 성공
Web nginx smoke docker run -p 18073:80 multi-tenant-local-web-admin:latest, docker run -p 18074:80 multi-tenant-local-web-wms:latest GET / 응답: web-admin 200, web-wms 200
CDK synth pnpm --filter @multi-tenant/infra-cdk synth 성공. ALB target group은 gateway-service만 포함합니다.
CDK BFF ingress CDK template에서 Admin/User BFF security group ingress 조회 Admin/User BFF로 들어오는 ingress는 gateway-service security group source 2개뿐입니다.
TypeScript pnpm --filter admin-bff-service typecheck, pnpm --filter user-bff-service typecheck 둘 다 통과

수정 범위

  • local compose에서 Admin/User BFF host port를 제거하고 web-admin/web-wms nginx 서비스를 추가했습니다.
  • Web Dockerfile이 루트 TypeScript 설정을 복사하고 Vite API base URL을 build arg로 받을 수 있게 수정했습니다.
  • Postman과 로컬 시스템 테스트 시나리오에서 BFF 직접 호출을 gateway 경유 호출로 정리했습니다.
  • CDK는 수정 없이 synth와 산출물 검사 기준으로 gateway-only 외부 노출 구조가 유효함을 확인했습니다.
